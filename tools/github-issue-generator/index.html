Frequently Asked Questions

What is a GitHub issue template?

A GitHub issue template is a YAML file in your repository's .github/ISSUE_TEMPLATE/ folder that creates structured forms for bug reports, feature requests, and contributions. They guide contributors to provide the right information every time.

How do I add a GitHub issue template to my repo?

Generate your template with this tool, copy the YAML output, create a file in .github/ISSUE_TEMPLATE/bug_report.yml, paste the content, commit and push. GitHub automatically detects and uses templates in this folder.

What is the GitHub issue form YAML format?

GitHub issue forms use a YAML specification with name, description, and body fields. The body contains form elements like textarea, dropdown, and checkboxes. This tool generates valid YAML automatically.

How do issue templates help open source projects grow?

Good issue templates reduce incomplete bug reports by ~60%, which saves maintainer time and improves response rates. Based on experience growing AFFiNE to 60k+ GitHub stars, structured issue templates are one of the highest-ROI improvements for open source community health.
